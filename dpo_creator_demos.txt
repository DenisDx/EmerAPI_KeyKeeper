<H3>Пример 2</H3>
<H4>Решаемая задача</H4>
Antifake: Создание полного набора документов в папку, без генерации в блокчейне
<H4>Описание</H4>
Предполагается, что Вы - владелец объекта \"бренд\", и этот бренд принадлежит Вашему адресу.<br>
Кроме этого, существует объект Antifake \"лот\", который также принадлежит Вашему адресу.<br>
<br>
Будут выбраны первый объекты из списка в качестве примера.<br>
Если у Вас нет ни одного лота или ни одного бренда, пример не сможет быть запущен.<br>
<br>
Все документы будут загружены в папку с именем \"demo2\", которая будет создана в папке с приложением.<br>
Вы можете изменить это, указав желаемую папку.<br>
<br>
Будут созданы:
<ul>
<li><b>private.csv</b> со всей информацией, необходимой для генерации. Далее записи в блокчейне смогут быть созданы на базе этого файла.</li>
<li><b>public.csv</b> - файл для загрузки на сервер antifake. В нем не содержится секретной информации</li>
<li><b>print.csv</b> - файл для отправки в типографию. В нем содержится вся секретная информация, необходимая для создания этикеток</li>
<li>Папка <b>QR1</b> будет содержать QR-коды, предназначенные для открытой части этикетки товара. Все данные для генерации кода также содержатся в файле print.csv, файлы могут быть предоставлены для удобства типографии</li>
<li>Папка <b>QR2</b> будет содержать QR-коды, предназначенные для скрытой части этикетки товара. Все данные для генерации кода также содержатся в файле print.csv, файлы могут быть предоставлены для удобства типографии</li>
<li>если в процессе создания возникнут ошибки, они будут сохранены в файле <b>errors.csv</b> - файл для отправки в типографию. </li>
</ul>
<H4>Дополнительная информация</H4>
Данный файл рассчитан на двухступенчатую генерацию (записи  не создаются в блокчейн сразу, а могут быть быть созданы позднее на основе файла private.csv<br>
Для защиты от возможных сквотеров имена будут иметь случайный суффикс (включена опция <font color=green>\"Добавлять случайны суффикс к имени\"</font>).<br>
Тем не менее, будет проводится проверка существования каждой записи в блокчейне и изменение суффикса в случае, если такое имя уже есть. Эта проверка требует времени, Вы можете отключить ее для ускорения процесса (отключив функцию <font color=green>\"проверять, свободно ли имя\"</font>)<br>
Если будет принято решение создавать записи в блокчейне сразу (будет включена опция <font color=green>\"Создавать записи в блокчейн\"</font> - генерацию случайного суффикса можно отключить.<br>
